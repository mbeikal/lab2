Лабораторна робота №1
Створення REST API для екологічних даних на базі Express.js та MongoDB
Мета роботи

Ознайомитися з основами створення серверної частини вебзастосунку на Node.js із використанням фреймворку Express.js та бази даних MongoDB.
Розробити REST API для зберігання, обробки та отримання екологічних показників (якості повітря та води) з реалізацією CRUD-операцій.

---

Використані технології

Node.js — середовище виконання JavaScript

Express.js — фреймворк для створення REST-API

MongoDB + Mongoose — документоорієнтована база даних

dotenv — зберігання змінних середовища

morgan — логування HTTP-запитів

cors — підтримка міждоменної взаємодії (CORS)

---

Опис функціональності

1. Модель AirQuality

Містить дані про якість повітря:

stationId, city, timestamp, location

pollutants (PM2.5, PM10, NO₂, SO₂, CO, O₃)

weather (температура, вологість)

2. Модель WaterQuality

Містить показники якості води:

stationId, body, city, timestamp, location

metrics (pH, кисень, каламутність, нітрати)

heavyMetals (свинець, миш’як, ртуть)

---

3. Реалізовано CRUD-операції
   Операція Метод Ендпойнт
   Отримати всі дані GET /api/air-quality
   Отримати за ID GET /api/air-quality/:id
   Створити новий запис POST /api/air-quality
   Оновити запис PUT /api/air-quality/:id
   Видалити запис DELETE /api/air-quality/:id

---

Результати тестування

Отримано успішну відповідь для всіх CRUD-операцій.

Дані зберігаються у MongoDB.

Сервер повертає коректні JSON-результати.

Працює фільтрація за city, stationId, timestamp, підтримується сортування й пагінація.
